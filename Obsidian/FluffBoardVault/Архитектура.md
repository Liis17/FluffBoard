# Архитектура

Parent: [[Index]]

## Tech Stack

| Область | Технологии | Роль |
|---------|------------|------|
| Frontend | React 19, React DOM, Vite 8, JavaScript | Одностраничный пользовательский интерфейс и локальная разработка |
| Backend | ASP.NET Core minimal API, .NET 10 | HTTP API и CORS-политика |
| Качество | Oxlint | Линтинг frontend-кода |

## Компоненты

- [[backend/API]] — запускается на `http://localhost:5279` в development-профиле и предоставляет `GET /api/hello`.
- [[frontend/Приложение]] — Vite dev server, обычно `http://localhost:5173`; отрисовывает результат запроса к backend.

## Сквозной поток: приветствие API

1. `frontend/src/main.jsx` монтирует React-компонент `App`.
2. `App` делает `fetch('http://localhost:5279/api/hello')` при первом рендере.
3. `backend/Program.cs` разрешает origin Vite через CORS и обрабатывает запрос.
4. Endpoint возвращает JSON с полем `message`; frontend показывает текст или сообщение о недоступности backend.

## Паттерны и соглашения

- Backend использует top-level statements и minimal API: маршрут объявляется через `app.MapGet(...)` в `Program.cs`.
- Frontend использует функциональные React-компоненты и hooks: `useState` хранит состояние, `useEffect` запускает запрос.
- Статическая стилизация находится рядом с React-кодом: `App.css` для экрана и `index.css` для глобальных правил.

## Как добавить компонент

1. Добавь реализацию в фактический домен `backend` или `frontend`.
2. Свяжи её с точкой входа или маршрутом и проверь соответствующую команду запуска.
3. Создай либо обнови заметку в этой папке vault и добавь ссылку в [[Index]].
4. Если меняется взаимодействие клиент—сервер, актуализируй этот документ.
