## Tech Stack

| Frontend | React 19, React DOM, Vite 8, JavaScript | Одностраничный пользовательский интерфейс и локальная разработка |

## Компоненты

- [[backend/API]] — запускается на `http://localhost:5279` в development-профиле и предоставляет `GET /api/hello` и API GitHub issues.
- [[frontend/Приложение]] — Vite dev server, обычно `http://localhost:5173`; получает и отрисовывает issues выбранного репозитория.

## Сквозной поток: GitHub issues

1. Пользователь вводит owner и repository в `App` и отправляет форму.
2. `App` делает запрос `GET /api/github/repos/{owner}/{repository}/issues` к backend.
3. `backend/Program.cs` разрешает origin Vite через CORS и передаёт запрос `GitHubClient`.
4. `GitHubClient` загружает все страницы GitHub REST API, исключает pull request'ы и возвращает нормализованные issue, labels и assignees.
5. `App` отображает список либо понятную ошибку.

## Паттерны и соглашения

- Backend использует top-level statements и minimal API: маршрут объявляется через `app.MapGet(...)` в `Program.cs`.
- Токен GitHub не передаётся из браузера: backend читает его из `GitHub__Token` и добавляет к исходящему запросу, только если он задан.
- Frontend использует функциональные React-компоненты и hooks: `useState` хранит поля формы, состояние загрузки и список issues; submit handler запускает запрос.
- Статическая стилизация находится рядом с React-кодом: `App.css` для экрана и `index.css` для глобальных правил.

## Как добавить компонент

1. Добавь реализацию в фактический домен `backend` или `frontend`.
2. Свяжи её с точкой входа или маршрутом и проверь соответствующую команду запуска.
3. Создай либо обнови заметку в этой папке vault и добавь ссылку в [[Index]].
4. Если меняется взаимодействие клиент—сервер, актуализируй этот документ.
