# Приложение

Parent: [[Index]]

## Назначение

React-клиент FluffBoard показывает защищённую Kanban-доску одной репозитории GitHub. Пользователь входит по локальным учётным данным, затем создаёт и редактирует задачи; каждая запись сразу отправляется в GitHub.

## Файлы

- `frontend/src/App.jsx` — экран входа, API-клиент с cookie, Kanban-колонки, карточки и редактор задач.
- `frontend/src/App.css` — стили логина, доски и бокового редактора.
- `frontend/vite.config.js` — React-плагин и proxy `/api` на backend development server.

## Ключевые функции

| Функция | Описание |
|---|---|
| `api(...)` | Выполняет запрос с cookie-сессией, разбирает API ошибки. |
| `App()` | Восстанавливает сессию, загружает issues/labels/users и управляет редактором. |
| `LoginScreen(...)` | Вход по логину и паролю из конфигурации backend. |
| `TaskEditor(...)` | Создаёт или редактирует title, body, status, labels и исполнителя. |
| `getWorkflowStatus(...)` | Раскладывает GitHub issue по `todo`, `in-progress`, `done`; закрытые issues считаются готовыми. |

## Важные детали

Клиент не выбирает репозиторий и не получает GitHub token: репозиторий и сервисный токен заданы server-side. Development proxy делает браузерный запрос относительным (`/api/...`), поэтому cookie не требует отдельной CORS-настройки Vite-клиента.
